Detailed Description

Forward Rule Engine Use construct {} where {} SPARQL Query as Rule Optimizations: Do not create Mappings, create triples directly Consider rules for which new relevant triples are available Consider solutions with new triple Focus on new triples using specific Graph Index sorted by timestamp Eval transitive rule at saturation using specific Java code Eval pseudo transitive rule just after it's transitive rule (cf rdf:type & rdfs:subClassOf)
